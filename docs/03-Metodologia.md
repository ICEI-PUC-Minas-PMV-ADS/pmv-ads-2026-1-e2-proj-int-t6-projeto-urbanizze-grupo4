
# Metodologia

A metodologia de trabalho do grupo para o projeto **Urbanizze** baseia-se em processos ágeis e no uso de ferramentas colaborativas que permitem a integração contínua e a comunicação eficiente entre os membros, garantindo o foco na resolução de problemas urbanos conforme as diretrizes da ODS 11.

## Controle de Versão

A ferramenta de controle de versão adotada no projeto foi o [Git](https://git-scm.com/), sendo que o [GitHub](https://github.com) foi utilizado para hospedagem do repositório.

O projeto segue a seguinte convenção para o nome de branches:

* **main**: versão estável já testada do software.
* **unstable**: versão já testada do software, porém instável.
* **testing**: versão em testes do software.
* **dev**: versão de desenvolvimento do software.

Quanto à gerência de issues, o projeto adota a seguinte convenção para etiquetas:

* **documentation**: melhorias ou acréscimos à documentação.
* **bug**: uma funcionalidade encontra-se com problemas.
* **enhancement**: uma funcionalidade precisa ser melhorada.
* **feature**: uma nova funcionalidade precisa ser introduzida.

A configuração do projeto foi realizada criando-se um repositório central no GitHub. A gerência de commits é feita de forma frequente para evitar grandes conflitos. As merges são realizadas preferencialmente via *Pull Requests* após a revisão básica dos pares, garantindo que a branch `main` contenha apenas código funcional.

### Processo

Para o gerenciamento do projeto, o grupo utiliza o **GitHub Projects**, onde foi estruturado um quadro **Kanban** para acompanhamento visual das tarefas. O processo consiste em:

1. **Backlog:** Todas as funcionalidades e requisitos levantados durante o Design Thinking.
2. **To Do:** Tarefas selecionadas para a sprint ou semana atual.
3. **In Progress:** Tarefas que os membros já iniciaram o desenvolvimento.
4. **Done:** Tarefas concluídas, testadas e com documentação atualizada.

As reuniões de alinhamento são realizadas semanalmente para revisar o que foi feito e planejar os próximos passos.

### Ferramentas

As ferramentas empregadas no projeto são:

* **Editor de código (VS Code):** Escolhido por ser leve, possuir integração nativa com o Git e suporte extensivo a JavaScript.
* **Ferramentas de comunicação (WhatsApp/Teams):** Utilizadas para comunicações rápidas e reuniões de alinhamento síncronas.
* **Ferramentas de desenho de tela (Figma):** Empregada para a criação de wireframes e protótipos de alta fidelidade.
* **GitHub:** Utilizado tanto para o versionamento do código quanto para a gestão de tarefas (GitHub Projects) e documentação.
