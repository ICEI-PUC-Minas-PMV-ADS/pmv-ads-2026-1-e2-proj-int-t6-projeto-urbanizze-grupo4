# Plano de Testes de Usabilidade

Este plano de testes refere-se a um sistema de gestão de denúncias desenvolvido para auxiliar a população no registro e acompanhamento de problemas urbanos, especialmente aqueles relacionados a riscos causados por eventos climáticos, como enchentes e deslizamentos. A proposta surge a partir de situações recentes enfrentadas por cidades mineiras, como Ubá e Juiz de Fora, buscando melhorar a comunicação entre cidadãos e autoridades públicas.
## Definição do(s) objetivo(s)

Antes de iniciar os testes, é essencial definir o que se deseja avaliar na usabilidade do sistema.
Neste projeto, os objetivos são:
    • Verificar se os usuários conseguem utilizar o sistema de forma intuitiva e sem dificuldades significativas.
    • Avaliar a facilidade de cadastro e autenticação.
    • Identificar possíveis dificuldades no processo de criação de denúncias.
    • Avaliar a eficiência da comunicação entre cidadãos e órgãos públicos.
    • Verificar a compreensão dos diferentes perfis de acesso (cidadão e funcionário).
    • Testar a usabilidade das funcionalidades de busca, filtros e notificações.
## Seleção dos participantes

Para garantir que o teste reflita o uso real do sistema, foram definidos participantes representativos do público-alvo.
Critérios para selecionar participantes:
    • Usuários comuns (cidadãos), incluindo moradores de áreas urbanas e regiões de risco.
    • Funcionários vinculados a prefeituras ou departamentos responsáveis pela análise de denúncias.
    • Diferentes níveis de familiaridade com tecnologia.

    Quantidade recomendada:
Mínimo: 5 participantes.
Ideal: Entre 8 e 12 participantes.
## Definição de cenários de teste

Os cenários representam tarefas reais que os usuários executam no sistema. Neste projeto, foram definidos cinco cenários principais:
Cenário 1: Primeiro acesso (Cidadão)
Objetivo: Avaliar funcionalidades de cadastro, login e recuperação de senha.
Contexto: Um novo usuário acessa a plataforma pela primeira vez e precisa criar uma conta para utilizar o sistema.
Tarefa(s):
    • Criar uma conta no sistema.
    • Realizar login.
    • Simular recuperação de senha.
Critério(s) de Sucesso(s):
    • O usuário consegue criar a conta sem dificuldades.
    • O login é realizado corretamente.
    • O processo de recuperação de senha é compreendido.
    • As ações são concluídas sem necessidade de ajuda.

Cenário 2: Registro de denúncia
Objetivo: Avaliar o processo de criação de denúncias.
Contexto: O usuário deseja registrar um problema urbano, como alagamento ou risco estrutural.
Tarefa(s):
    • Registrar uma denúncia.
    • Inserir descrição detalhada.
    • Adicionar foto ou anexo.
Critério(s) de Sucesso(s):
    • A denúncia é registrada corretamente.
    • O usuário compreende os campos obrigatórios.
    • O envio de anexos ocorre sem erros.
    • O processo é concluído de forma rápida e clara.

Cenário 3: Interação em denúncia
Objetivo: Avaliar comunicação e notificações.
Contexto: O usuário deseja acompanhar uma denúncia já registrada.
Tarefa(s):
    • Acessar uma denúncia existente.
    • Enviar uma mensagem.
    • Verificar notificações.
Critério(s) de Sucesso(s):
    • O usuário consegue acessar a denúncia.
    • A mensagem é enviada com sucesso.
    • As notificações são visualizadas corretamente.

Cenário 4: Busca e filtros
Objetivo: Avaliar funcionalidades de pesquisa.
Contexto: O usuário precisa localizar denúncias específicas.
Tarefa(s):
    • Buscar uma denúncia.
    • Aplicar filtros por status ou categoria.
Critério(s) de Sucesso(s):
    • A busca retorna resultados relevantes.
    • Os filtros funcionam corretamente.
    • O usuário encontra a informação desejada com facilidade.

Cenário 5: Acesso como funcionário
Objetivo: Avaliar funcionalidades administrativas.
Contexto: Um funcionário acessa o sistema para gerenciar denúncias.
Tarefa(s):
    • Acessar o sistema com perfil de funcionário.
    • Visualizar denúncias.
    • Responder uma denúncia.
    • Navegar entre departamentos.
Critério(s) de Sucesso(s):
    • O funcionário acessa corretamente o sistema.
    • As denúncias são visualizadas sem dificuldade.
    • As respostas são enviadas corretamente.
    • As diferenças entre perfis são compreendidas.
## Métodos de coleta de dados

Os dados coletados devem ajudar a entender a experiência dos usuários.
Serão utilizadas:
    • Métricas quantitativas:
        ◦ Taxa de sucesso nas tarefas
        ◦ Tempo de execução
        ◦ Número de erros
        ◦ Taxa de abandono
    • Métricas qualitativas:
        ◦ Nível de dificuldade percebida
        ◦ Clareza da interface
        ◦ Satisfação do usuário
Além disso, será utilizado o método Think Aloud (pensar em voz alta), com testes moderados, realizados presencialmente ou remotamente, com duração de 20 a 30 minutos por participante.
Também serão aplicadas perguntas pós-teste para coleta de feedback.
Para cada participante, os dados serão coletados respeitando a LGPD, sem identificação de informações sensíveis.
